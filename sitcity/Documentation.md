##  Project Overview

**SITCITY** is a responsive and interactive furniture showcase web application built with React. The project focuses on delivering a modern user experience through smooth animations, intuitive navigation, and dynamic UI components.

This project was completed as an **assigned internship project** for the **Frontend Engineer role** at [ArchNexus](https://www.archsaintnexus.com/).

---

##  Features

* **Interactive Furniture Display**
  Users can explore chairs and furniture items through engaging and responsive UI interactions.

* **Carousel & Slider Functionality**
  Smooth, responsive carousels implemented using **React Slick** for displaying furniture items, thumbnails, and reviews.

* **Smooth Animations**
  High-quality UI animations powered by **GSAP** to enhance visual flow and user engagement.

* **Modern Navigation UI**
  Custom navigation design with animated interactions (non-standard burger-style menu).

* **Pan & Interactive Display Effects**
  Interactive visual effects to create an immersive browsing experience.

* **Responsive Design**
  Fully optimized for desktop, tablet, and mobile screen sizes.

* **Clean Component-Based Architecture**
  Modular and reusable React components for maintainability and scalability.

##  Installation & Setup

Follow the steps below to run the project locally.

###  Clone the repository

```bash
git clone https://github.com/ThinHubbins/sitcity.git
```

###  Navigate into the project directory

```bash
cd sitcity
```

###  Install dependencies

```bash
npm install
```

###  Start the development server

```bash
npm start
```

The application will open automatically in your browser.

##  Challenges & Lessons Learned

One of the most challenging parts of this project was building the **Hero section**.

Due to its complex design, adapting it for mobile responsiveness became difficult. Early design decisions made for larger screens caused layout issues on smaller devices.

To resolve this, the Hero section had to be **rebuilt from scratch**. Although the visual changes were subtle, the refactor required significant time and effort.

**Key lessons learned:**

* Avoid design shortcuts during early stages
* Plan responsiveness from the start
* Clean structure saves time in the long run
